# Open-Jarvis 源码导览

这份文档面向人类读者，帮助快速判断“某个行为到底由哪层控制”。如果你是代码代理或要做最小路径修改，先读根目录的 AGENTS.md；如果你需要先理解整体实现，再决定从哪里下手，这份文档更合适。

## 1. 总体结构

- main：Electron 主进程，负责窗口生命周期、数据库、Agent 运行时、IPC、MCP、审批规则、工作区文件访问和嵌入式工具链。
- preload：唯一可信的渲染层桥接面。所有 Electron / Node 能力都应该经由 window.api 暴露。
- renderer：React 19 应用，承担三栏布局、对话、右侧面板、看板、文件预览、设置与各种配置对话框。
- resources/tooling：打包用的 bun / uv / Python 运行时。

## 2. 主进程的真实控制点

### 启动与注册

- src/main/index.ts：应用启动入口。初始化数据库、注册 approval / agent / threads / models / mcp / skills IPC，并在退出时清理 Agent 与 MCP 资源。

### Agent 与工具调用

- src/main/agent/runtime.ts：创建 DeepAgent、装配模型、恢复历史、挂接 checkpoint、拼接技能来源。
- src/main/agent/local-sandbox.ts：本地沙箱封装，统一文本读取与命令执行。
- src/main/agent/mcp-runtime.ts：把 MCP server 暴露成 LangChain tools。当前支持 stdio、streamable_http、sse，并按 server.id + 配置签名做连接缓存。
- src/main/agent/system-prompt.ts：系统提示词。默认行为不应散落在 UI 文案中修改。

### 线程、审批与配置

- src/main/db：线程与消息持久化。
- src/main/checkpointer/sqljs-saver.ts：每线程单独的 LangGraph checkpoint。
- src/main/approval-settings.ts：线程审批模式、工作区自动批准规则、工具签名归一化。
- src/main/mcp-config.ts：MCP server 的存储、导入、导出和线程启用状态。
- src/main/openai-compatible-profiles.ts：OpenAI-compatible 模型档案存储。
- src/main/skill-config.ts：全局与工作区技能目录解析。
- src/main/tooling.ts：运行时定位嵌入式工具链目录。

### IPC 实际入口

- src/main/ipc/approval.ts：approval:getMode、approval:setMode、approval:shouldAutoApprove。
- src/main/ipc/agent.ts：agent:invoke、resume、interrupt、cancel。
- src/main/ipc/threads.ts：线程创建、更新、删除、历史、标题。
- src/main/ipc/models.ts：模型、provider、API Key、OpenAI-compatible 档案，以及 workspace:get/set/select/readFile/readBinaryFile/openCurrentFolder。
- src/main/ipc/mcp.ts：MCP server 配置、导入导出、线程启用状态。
- src/main/ipc/skills.ts：工作区技能目录管理。

## 3. 渲染层的状态与布局

### 顶层骨架

- src/renderer/src/App.tsx：三栏布局和看板总览的分叉点。应用初始化时会先加载线程，没有线程时自动创建默认线程。

### 全局状态与线程状态

- src/renderer/src/lib/store.ts：全局 Zustand store。保存线程列表、当前线程、模型列表、设置弹窗、看板开关等。createThread 会继承当前线程的 model、workspacePath、approvalMode。
- src/renderer/src/lib/thread-context.tsx：线程内状态源。消息、草稿、todo、工作区文件、MCP 启用列表、审批请求、token usage、打开文件标签都在这里。

### 主要 UI 面

- src/renderer/src/components/sidebar/ThreadSidebar.tsx：会话侧栏、新会话、重命名、批量删除、进入看板。
- src/renderer/src/components/chat/ChatContainer.tsx：发送消息、流式状态、审批状态接入。
- src/renderer/src/components/chat/ToolCallRenderer.tsx：工具调用卡片、待审批状态和批准/拒绝/编辑入口。
- src/renderer/src/components/chat/ContextUsageIndicator.tsx：上下文窗口使用情况，展示输入 / 输出 / 缓存 token。
- src/renderer/src/components/panels/RightPanel.tsx：右侧的任务、文件、子智能体三段式面板。
- src/renderer/src/components/kanban：多线程看板视图。
- src/renderer/src/components/tabs：中间区标签、文件查看器和媒体预览。

## 4. 当前几个容易找错地方的行为

### 新建会话为什么“没有新建”

如果当前线程没有消息且草稿为空，侧栏点击“新会话”会复用当前线程，而不是额外创建空线程。这个行为在 src/renderer/src/components/sidebar/ThreadSidebar.tsx。

### 删除最后一个会话为什么不会回到空白态

store.ts 的 deleteThreads() 会在删空后立即补一个新线程，避免 currentThreadId 变成 null 导致工作区文件面板失效。

### 审批规则存在哪里

线程默认审批模式存在 thread metadata。工作区级自动批准规则单独写到工作区目录下的 .open-jarvis/approval-rules.json，不在 .openwork 中。

### Token 统计为什么刷新后还在

thread-context.tsx 会把每个线程的 token usage 持久化到 localStorage，用于上下文窗口监控。

## 5. 嵌入式工具链与打包

- scripts/prepare-embedded-tooling.mjs 会下载并整理 uv、bun 和 Python。
- 脚本会重写 Python 安装中的符号链接，避免 macOS 打包后出现绝对链接指向临时目录，进而破坏签名或 notarization。
- package.json 中的 package:dir、dist、dist:mac、dist:win、dist:linux 都依赖 prepare:tooling。

## 6. 修改时的最小同步面

### 新增 IPC

1. 主进程 handler。
2. preload/index.ts。
3. preload/index.d.ts。
4. 渲染层调用点。

### 改工作区文件读取

1. 优先看 src/main/ipc/models.ts。
2. 保持 text-encoding.ts 与 local-sandbox.ts 的解码策略一致。

### 改审批逻辑

1. 先改 approval-settings.ts。
2. 再看 ipc/approval.ts 和渲染层审批 UI 是否需要跟着同步。

### 改 MCP 行为

1. mcp-config.ts 负责配置存储。
2. mcp-runtime.ts 负责传输实现与缓存。
3. MCPConfigDialog.tsx 负责用户配置界面。

## 7. 推荐验证

- 只改文档：至少确认 Markdown 链接和路径名称正确。
- 改渲染层：bun run typecheck:web。
- 改主进程 / preload：bun run typecheck:node。
- 提交前最少：bun run typecheck 和 bun run lint。
