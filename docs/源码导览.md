## 1. 总体架构

| 层 | 技术栈 | 职责 |
|---|---|---|
| **main** | TypeScript + Electron 42 + electron-vite | 窗口生命周期、数据库、Agent 运行时、IPC、MCP、审批规则、工作区文件访问、嵌入式工具链 |
| **preload** | TypeScript + contextBridge | 唯一可信的渲染层桥接面，所有 Electron / Node 能力经由 `window.api` 暴露 |
| **renderer** | React 19 + Tailwind CSS 4 + Radix UI + Zustand | 三栏布局、对话、右侧面板、看板、文件预览、设置与各种配置对话框 |
| **resources/tooling** | — | 打包用的 bun / uv / Python 运行时 |

**核心依赖关系：**

```
renderer ──(window.api)──> preload ──(ipcMain/ipcRenderer)──> main
                                    ──(LangGraph SDK useStream)──> agent runtime
```

**状态分层：**

- 全局 UI 与线程列表 → `store.ts`（Zustand）
- 线程内消息、草稿、文件、todo、subagent、审批态、token usage → `thread-context.tsx`

## 3. 主进程的真实控制点

### 3.1 启动与注册

**`src/main/index.ts`**（263 行）

- 创建 BrowserWindow：macOS 使用 `hiddenInset` 标题栏样式，`trafficLightPosition: { x: 16, y: 14 }` 与渲染层首行 32px 标题栏对齐。
- 窗口尺寸持久化到 electron-store（`windowBounds`、`windowWasMaximized`、`windowLaunchedBefore`），首次启动自动最大化。
- 初始化 SQLite 数据库 `initializeDatabase()`。
- **`await syncBundledSkillsToGlobalRoot()`**：将 `skills/`（开发）或 `default-skills`（打包）种子同步到 `~/.open-jarvis/skills/`。
- 注册 **7 组** IPC handler：approval / agent / threads / models / **mcp** / skills / **settings**。
- **`bootstrapMCPServers(getMCPServers())`**：启动时预连接已启用 MCP 服务器。
- 退出前调用 `closeAllRuntimeResources()`，清理所有 checkpointer 与 MCP 连接。
- 开发模式支持 F12 切换 DevTools，HMR 通过 `ELECTRON_RENDERER_URL` 环境变量。

### 3.2 Agent 运行时

**`src/main/agent/runtime.ts`**（1337 行）

核心函数 `createAgentRuntime(options)`，组装流程：

1. **模型实例化**（`getModelInstance`）：
   - `claude*` → `ChatAnthropic`
   - `gpt*` / `o1*` / `o3*` / `o4*` → `ChatOpenAI`
   - `gemini*` → `ChatGoogleGenerativeAI`
   - `oac:*` → 自定义 `OpenAICompatibleChatCompletions`（继承 `ChatOpenAICompletions`，重写 `completionWithRetry` 做消息归一化）
2. **Checkpoint 绑定**：每线程独立 `SqlJsSaver`，缓存在 `checkpointers` Map 中。
3. **沙箱创建**：`LocalSandbox`，`virtualMode: false`（使用绝对路径），超时 120s，最大输出 100KB。
4. **System Prompt 拼接**：`getSystemPrompt(workspacePath)` = 工作区路径段 + `BASE_SYSTEM_PROMPT`。
5. **MCP 工具收集**：从线程 metadata 读取 `enabledMcpServerIds`，调用 `getMCPToolsForServers()`。
6. **HITL 中断配置**：默认对 `execute` + 所有已启用 MCP 工具名设置 `interruptOn`。
7. **Skill 源解析**：`resolveSkillSourcesForWorkspace(workspacePath)`。
8. **创建 Agent**：`createDeepAgent({ model, checkpointer, backend, systemPrompt, filesystemSystemPrompt, interruptOn, tools, skills })`。

**`src/main/agent/system-prompt.ts`**（180 行）

`BASE_SYSTEM_PROMPT` 常量，定义 Agent 默认行为约束：简洁回复、文件分页读取、工作区路径处理、工具审批要求、todo 管理、子代理委派、HITL 拒绝处理。**改智能体默认策略时，优先改这里，不要散落在 UI 文案中。**

**`src/main/agent/local-sandbox.ts`**（1553 行）

`LocalSandbox` 继承 deepagents 的 `FilesystemBackend`，实现 `SandboxBackendProtocolV2`：

- Shell 命令执行：`spawn` 子进程，注入嵌入式 bun/uv/Python 运行时 PATH。
- 文件读取：`decodeTextBuffer()` 统一 UTF-8/GB18030 解码，带行号标注。
- 二进制/文本 MIME 检测。
- macOS 上 `.doc/.docx/.odt/.rtf` 通过 `textutil` 提取文本。
- `buildWorkspaceRuntimeCommand()`：构建 shell 前导，设置 PATH、虚拟环境、npm/pnpm/yarn → bun 重定向。

**`src/main/agent/mcp-runtime.ts`**（594 行）

MCP 连接管理，使用 `@modelcontextprotocol/sdk` 客户端：

- 三种传输：`stdio`、`streamable_http`、`sse`。
- 工具名归一化：`{server_prefix}_{tool_name}`；连接缓存按 `server.id + 配置签名` 失效。
- `bootstrapMCPServers()`：应用启动及 MCP 配置变更后预连接。
- `closeAllMCPConnections()` 用于应用退出清理。

### 3.3 线程、存储与配置

**`src/main/db/index.ts`**（260 行）

SQLite 数据库（sql.js，纯 JS 无原生依赖），三张表：

- `threads`：thread_id, created_at, updated_at, metadata, status, thread_values, title
- `runs`：run_id, thread_id, assistant_id, status, metadata, kwargs
- `assistants`：assistant_id, graph_id, model, config

磁盘写入 100ms 防抖。

**`src/main/checkpointer/sqljs-saver.ts`**（511 行）

`SqlJsSaver` 继承 `@langchain/langgraph-checkpoint` 的 `BaseCheckpointSaver`。每线程独立 SQLite 数据库，100MB 文件大小限制 + 自动备份，100ms 防抖磁盘持久化。

**`src/main/storage.ts`**（143 行）

管理 `~/.open-jarvis` 目录（含 `~/.openwork` 旧目录迁移）。API Key 存储为 `.env` 文件格式，环境变量名：`ANTHROPIC_API_KEY`、`OPENAI_API_KEY`、`GOOGLE_API_KEY`。

**`src/main/approval-settings.ts`**（160 行）

- 线程级审批模式：`manual` / `auto`，存于线程 metadata。
- 工作区级自动批准规则：存于 `.open-jarvis/approval-rules.json`。
- `createApprovalSignature()`：对工具参数归一化（execute 命令：空白归一 + 数字替换；其他工具：排序 JSON），避免命令细节波动导致规则失配。

**`src/main/mcp-config.ts`**（197 行）

MCP server 配置 CRUD，存储在 electron-store 的 `mcpServers` 键下。支持 JSON 导入导出。

**`src/main/openai-compatible-profiles.ts`**（56 行）

自定义 OpenAI 兼容模型档案 CRUD，存储在 electron-store 的 `openaiCompatibleProfiles` 键下。模型 ID 使用 `oac:` 前缀。

**`src/main/skill-config.ts`**（75 行）

- 运行时 Agent 始终读 `~/.open-jarvis/skills/`（全局、与工作区无关）。
- `getBundledSkillsRoot()`：开发态 `../../skills`，打包态 `process.resourcesPath/default-skills`。
- `syncBundledSkillsToGlobalRoot()`：仅当目标子目录不存在时复制，不覆盖用户修改。

**`src/main/global-config.ts`**（209 行）

- 全局 JSON 配置 import/export（OpenAI 兼容 profile、MCP、代理、默认模型、skills）。

**`src/main/memory-settings.ts`**（54 行）

- `memoryConsolidationEnabled`（默认 **false**）、`skillPromotionRecallThreshold`（默认 3）。

**`src/main/text-encoding.ts`**（27 行）

`decodeTextBuffer()`：UTF-8 优先解码，若出现替换字符（无效 UTF-8），回退到 GB18030（iconv-lite）。`toWellFormedText()` 清理孤立代理对。**Windows 中文编码兼容必须统一走这里。**

**`src/main/tooling.ts`**（72 行）

嵌入式工具链运行时定位：
- 打包版 → `process.resourcesPath/tooling/<platform-arch>`
- 开发态 → 仓库内 `resources/tooling/<platform-arch>`

返回 `EmbeddedToolingRuntime`：uv、bun、python、binDir 路径 + manifest 数据。

**`src/main/services/title-generator.ts`**（52 行）

启发式标题生成：短消息直接使用，长消息截断至约 50 字符 + 省略号。不调用 LLM。

**`src/main/services/workspace-watcher.ts`**（135 行）

`fs.watch` 递归监听工作区目录，500ms 防抖，跳过隐藏文件和 node_modules，通过 `workspace:files-changed` IPC 事件推送至渲染层。

**`src/model-context.ts`**（78 行）

模型上下文窗口大小查找表：

| 模型族 | 上下文窗口 |
|---|---|
| Claude 系列 | 200K |
| GPT-5 | 400K |
| GPT-4.1 | 1M |
| GPT-4o / GPT-4-turbo | 128K |
| o1 / o3 / o4 | 200K |
| Gemini 3 Pro / 2.5 Pro / 1.5 Pro | 2M |
| Gemini 其他 | 1M |
| DeepSeek | 64K |
| Qwen | 131K |
| GLM / ChatGLM | 128K |
| MiniMax / ABAB | 1M |
| 默认 | 128K |

支持 OpenAI-compatible 档案级 `contextWindow` 显式覆盖。

### 3.4 IPC 入口

**`src/main/ipc/agent.ts`**（858 行）

- `agent:invoke`：ipcMain.on 流式调用，活跃运行跟踪在 `activeRuns` Map（每线程一个 AbortController）。
- `agent:resume`：HITL 审批后恢复运行。
- `agent:interrupt`：批准/拒绝中断响应。
- `agent:cancel`：ipcMain.handle 取消运行。
- 流式通道：`agent:stream:${threadId}`，双模式（`messages` + `values`）。

**`src/main/ipc/approval.ts`**（43 行）

- `approval:getMode`：获取线程审批模式。
- `approval:setMode`：设置线程审批模式。
- `approval:shouldAutoApprove`：检查工作区自动批准规则。

**`src/main/ipc/threads.ts`**（336 行）

线程完整 CRUD：list / get / create / update / delete / deleteMany / history（checkpoint 列表）/ rewindToMessage（截断 checkpoint 历史到指定用户消息）/ generateTitle。删除线程时同步关闭并移除 checkpoint 文件。

**`src/main/ipc/models.ts`**（571 行）

最大的 IPC 文件，同时承载模型与工作区两类职责：

- **模型**：list / getDefault / setDefault / listProviders / setApiKey / getApiKey / deleteApiKey / openaiCompatibleList / openaiCompatibleUpsert / openaiCompatibleDelete。
- **工作区**：workspace:get / set / select / loadFromDisk / readFile / readBinaryFile / openCurrentFolder。
- 内置 18 个模型配置（Claude 4.5 系列、GPT-5/o 系列、Gemini 3/2.5 系列），默认模型 `claude-sonnet-4-5-20250929`。

**`src/main/ipc/mcp.ts`**（143 行）

MCP server CRUD：listServers / upsertServer / deleteServer / importServers / exportServers / getEnabledForThread / setEnabledForThread。线程级 MCP 启用状态存于线程 metadata。

**`src/main/ipc/skills.ts`**（425 行）

Skill 管理：listSources / setSources / listWorkspaceSkillFolders / importFolder（对话框选择）/ createSkill（创建 SKILL.md 含 YAML frontmatter）/ readSkillMarkdown / writeSkillMarkdown / renameSkillFolder / deleteSkillFolders。

**`src/main/ipc/settings.ts`**（539 行）

代理、记忆、全局配置、工具链版本、桌面通知：
- 代理：getProxyConfig / setProxyConfig（设置后自动调用 applyGlobalProxyDispatcher）。
- 记忆：getMemorySettings / setMemorySettings / listWorkspaceMemories / getWorkspaceMemoryDocument / updateWorkspaceMemoryDocument / deleteWorkspaceMemoryDocument。
- 全局配置：exportGlobalConfigToFile / importGlobalConfigFromFile（JSON 导入导出）。
- 工具链版本：getToolingVersions。
- 桌面通知：showDesktopNotification（Notification API，点击恢复窗口）。

## 5. 渲染层的状态与布局

### 5.1 入口与顶层骨架

**`src/renderer/src/main.tsx`**

React 19 StrictMode 入口。导入 `./lib/locales` 初始化 i18n；主题从 localStorage `openwork-theme` 读取。

**`src/renderer/src/App.tsx`**（209 行）

- `App` 组件：初始化时 `loadThreads()`，无线程则自动创建默认线程。
- `AppShell`：三栏可调整布局：
  - 左侧栏：220px 默认，min 220，max 350
  - 中间主区：弹性宽度
  - 右侧面板：280px 默认，min 230，max 450
- macOS 显示 `WindowTitleBar`。
- 看板视图与对话+标签+右侧面板布局互斥切换。
- 设置对话框通过 `SettingsDialog` 统一入口。

**`src/renderer/index.html`**

`lang="zh-CN"`，`class="dark"` 默认，严格 CSP 策略（default-src 'self'; img-src 'self' data: blob:; media-src 'self' blob: data:; frame-src 'self' blob: data:）。

### 5.2 全局状态与线程状态

**`src/renderer/src/lib/store.ts`**（368 行）

Zustand 全局 store `useAppStore`：

| 状态 | 说明 |
|---|---|
| `threads` / `currentThreadId` | 线程列表与当前选中 |
| `models` / `providers` | 模型与提供商列表 |
| `language` | UI 语言（`openwork-language`，默认 zh-CN） |
| `notificationSound` 等 | 桌面通知与提示音偏好 |
| `rightPanelTab` | 右侧面板当前标签（todos / files / subagents） |
| `settingsOpen` | 设置弹窗开关 |
| `sidebarCollapsed` | 侧栏折叠状态 |
| `showKanbanView` / `showSubagentsInKanban` | 看板视图开关 |
| `colorMode` | 亮色/深色模式（持久化到 localStorage `openwork-theme`） |

关键行为：
- `createThread()` 继承当前线程的 `model`、`workspacePath`、`approvalMode`。
- `deleteThreads()` 删空后立即补一个新线程，避免 `currentThreadId === null` 导致工作区面板悬空。

**`src/renderer/src/lib/thread-context.tsx`**（1616 行）

最复杂的渲染层文件。`ThreadProvider` 管理每线程状态：

| 状态 | 说明 |
|---|---|
| `messages` | 对话消息列表 |
| `draftInput` | 输入框草稿 |
| `todos` | 任务列表 |
| `workspaceFiles` | 工作区文件列表 |
| `workspacePath` | 工作区路径 |
| `enabledMcpServerIds` | 已启用 MCP server ID |
| `subagents` | 子智能体状态 |
| `pendingApproval` | 待审批工具调用 |
| `currentModel` | 当前模型 |
| `openFiles` / `activeTab` | 打开的文件标签 |
| `fileContents` | 文件内容缓存 |
| `tokenUsage` | Token 使用量（持久化到 localStorage） |
| `approvalMode` | 审批模式 |
| `interruptionQueue` | 插嘴队列（AI 流式响应期间用户发送的消息） |

每个活跃线程挂接 `ThreadStreamHolder`，创建 `ElectronIPCTransport` 并调用 LangGraph SDK 的 `useStream()`。

**`src/renderer/src/lib/electron-transport.ts`**（1537 行）

`ElectronIPCTransport` 实现 LangGraph SDK 的 `UseStreamTransport` 接口，是 Electron IPC 与 LangGraph 流格式的关键桥接层：

- 处理双流模式（messages + values）。
- 解析 LangChain 序列化格式（`lc: 1, type: "constructor", id: [...], kwargs: {...}`）。
- 从 AIMessageChunk / ToolMessage 提取内容。
- 流式工具调用累积。
- 子智能体（task）工具调用检测与完成追踪。
- HITL 中断处理（legacy + 新 langchain 格式 actionRequests/reviewConfigs）。
- Token usage 自定义事件发射（来自 usage_metadata）。

### 5.3 左侧会话栏

**`src/renderer/src/components/sidebar/ThreadSidebar.tsx`**

会话列表、新建、重命名、批量删除、切换到看板。当前逻辑：点击"新会话"时，如果当前线程仍为空线程（无消息、无草稿），会复用当前线程，不再额外创建新的空会话。

### 5.4 对话与消息

| 组件 | 职责 |
|---|---|

## 8. 当前几个容易找错地方的行为

### 新建会话为什么"没有新建"

如果当前线程没有消息且草稿为空，侧栏点击"新会话"会复用当前线程，而不是额外创建空线程。这个行为在 `src/renderer/src/components/sidebar/ThreadSidebar.tsx`。

### 删除最后一个会话为什么不会回到空白态

`store.ts` 的 `deleteThreads()` 会在删空后立即补一个新线程（继承当前线程的 model/workspacePath/approvalMode），避免 `currentThreadId` 变成 null 导致工作区文件面板失效。

### 审批规则存在哪里

线程默认审批模式存在 thread metadata。工作区级自动批准规则单独写到工作区目录下的 `.open-jarvis/approval-rules.json`，不在 `.openwork` 中。

### Token 统计为什么刷新后还在

`thread-context.tsx` 会把每个线程的 token usage 持久化到 localStorage，用于上下文窗口监控。

### 工作区 IPC 为什么在 models.ts 里

`src/main/ipc/models.ts` 同时承载模型和工作区两类 IPC handler，这是历史原因。工作区相关的 `workspace:get/set/select/readFile/readBinaryFile/openCurrentFolder` 都在这里注册，不存在单独的 workspace IPC 文件。

### OpenAI-compatible 消息归一化

`runtime.ts` 中 `OpenAICompatibleChatCompletions` 重写了 `completionWithRetry`，对 array-format content、reasoning_content、thinking blocks、lone surrogate 做归一化处理，避免不同 OpenAI 兼容 API 的格式差异导致崩溃。

### 插嘴队列（Interruption Queue）

AI 流式响应期间，用户可以继续输入消息。这些消息进入 `interruptionQueue`（`thread-context.tsx`），以 `_queued: true` 标记，在 `MessageBubble` 中以半透明 + "排队中"角标展示。流结束后，`ChatContainer` 的 `prevLoadingRef` effect 自动将队列消息合并为一条发送给大模型。取消操作会清空队列。

## 10. 修改时的最小同步面

### 新增 IPC

1. 主进程 handler（`src/main/ipc/*.ts`）。
2. preload 暴露（`src/preload/index.ts`）。
3. preload 类型声明（`src/preload/index.d.ts`）。
4. 渲染层调用点。

### 改工作区文件读取

1. 优先看 `src/main/ipc/models.ts` 的 workspace handler。
2. 保持 `text-encoding.ts` 与 `local-sandbox.ts` 的解码策略一致。

### 改审批逻辑

1. 先改 `approval-settings.ts`。
2. 再看 `ipc/approval.ts` 和渲染层审批 UI（`ToolCallRenderer.tsx` + 对话区底部审批栏）是否需要跟着同步。
3. 若涉及"记住自动批准"，确认 `.open-jarvis/approval-rules.json` 写入逻辑。

### 改 MCP 行为

1. `mcp-config.ts` 负责配置存储。
2. `mcp-runtime.ts` 负责传输实现与缓存。
3. `settings/MCPConfigPanel.tsx` 负责用户配置界面。

### 改 Agent 行为或工具链

1. Agent 行为优先看 `runtime.ts` 与 `system-prompt.ts`。
2. 本地文件沙箱与编码问题看 `local-sandbox.ts` 与 `text-encoding.ts`。
3. 嵌入式 bun/uv/Python 相关问题看 `tooling.ts` 与 `prepare-embedded-tooling.mjs`。

### 改上下文窗口 / Token 统计

1. UI 展示在 `ContextUsageIndicator.tsx`。
2. 事件汇总与传递在 `electron-transport.ts`。
3. 每线程持久化与消费在 `thread-context.tsx`。
4. 模型上下文窗口查找表在 `model-context.ts`。

### 改新建会话行为

1. 先看 `ThreadSidebar.tsx`。
2. 再看 `store.ts` 的 `createThread()` 是否需要继承更多 metadata。
3. 只有涉及持久化字段时再下探 `ipc/threads.ts` 与 `db`。
4. 若删线程后出现"无工作区"回归，优先核对 `deleteThreads()` 的兜底新线程逻辑。

## 13. 协作注意点

- 不要绕过 preload 直接在渲染层引入 Electron / Node 能力。
- 不要在主进程与沙箱里各写一套文本解码逻辑，统一走 `text-encoding.ts`。
- 不要弱化 HITL、工作区权限与"只信任当前工作区"的安全提示。
- 如果只是在 UI 上新增一个按钮或状态，先找真正决策行为的组件，不要上来改 store 或 DB。
- 如果一个文件只是桥接层，优先继续追到真正计算或变更状态的代码处再动手。
- 改模型清单时，同步 `model-context.ts` 的上下文窗口查找表。
- 改 IPC 方法签名时，必须同步 `preload/index.d.ts` 类型声明。
